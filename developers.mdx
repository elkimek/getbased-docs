# Developer documentation

getbased is a native ES module web app with no app bundler. It still has npm tooling for tests, type checks, Playwright, and deployment helpers. This section is the canonical developer documentation, kept here so contributor docs and public docs stay in one Mintlify site.

**New here?** Start with [CONTRIBUTING.md](https://github.com/elkimek/get-based/blob/main/CONTRIBUTING.md) for setup, running locally, tests, and PR guidelines.

## Primary reference

Use this section as the public, human-readable implementation reference. Keep internal agent handoff notes and private deployment context out of the public docs.

## Current implementation audit

This section was moved from `get-based/dev-docs` into the Mintlify docs site. During the move, the docs were checked against the live app tree on this machine:

- the app still uses native browser ES modules without bundling the app source;
- the repository does use npm tooling and a Vercel catalog-fetch step, so “zero-bundler” should not be confused with tooling-free;
- Biology Scores currently expose 19 definitions including Biological Coherence;
- public user docs live in `/guides`, while implementation details live here under `/developers`;

When adding a user-visible feature, update both the relevant user guide and the matching developer page in this same docs repo. Do not create a second long-lived docs surface in the app repository.
